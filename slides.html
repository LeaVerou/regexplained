What we learned

  • Regexes match anywhere, unless restricted
  • Matches cannot intersect
  • Case sensitive by default, use the i flag to change

What we learned

  • Dot (.) matches anything, except line breaks
  • Escape metacharacters with a backslash

The 12 metacharacters

[ { ( ) \ ^ $ . | ? * +

What we learned

  • {n} = n times
  • {n,} = at least n times
  • {m,n} = at least m times but no more than n times

What we learned

  • * = {0,} , + = {1,} , ? = {0,1}
  • Careful of accidental zero length matches!

What we learned

  • Quantifiers are greedy
  • You can change that by adding a ? after them

What we learned

  • Brackets = set, range or a combination of both
  • Concatenate multiple ranges for a union
  • Most metacharacters don’t need escaping in []

What we learned

  • \w = [a-zA-Z0-9_], \d = [0-9], \s ≈ [\t\r\n ]
  • Combine to form more complex character classes

Counting words (roughly)

function wordCount(text){
	return text.match(/\w+/g).length;
}
function wordCount(text){
	return text.split(/\s+/).length;
}

Hex color

Examples: #abc, #f00, #BADA55, #C0FFEE

Hex color

/^#[a-f\d]{3,6}$/i.test(str); // Wrong!
/^#([a-f\d]{3}){1,2}$/i.test(str);

What we learned

  • ^ negates a character class
  • \W = [^\w], \D = [^\d], \S = [^\s]
  • Even the dot is a character class: . = [^\r\n]

Strip HTML

function stripHTML(str){
    return str.replace(/<.+?>/g, '');
}
function stripHTML(str){
    return str.replace(/<[^>]+>/g, '');
}

Warning: Will fail in edge cases

What we learned

  • | for alternatives
  • Group with parenthses (Useful for alternatives or quantifiers)
  • Parentheses have special meaning

What we learned

  • Parentheses form capturing groups
  • Add ?: in the begninning to avoid this

What we learned

  • ^ = beginning of string
  • $ = end of string
  • beginning/end of lines, with the m flag

Number

Without exponent or digit separators

Examples: -1, .05, +1000, 3.1415926535, 42.

Number

/^[-+]?[\d.]+$/ // Too lax
/^[-+]?\d*\.?\d+$/ // False negatives: 5.
/^[-+]?\d*\.?\d*$/ // False positives: ., +., + etc
// Accurate, but is it worth it?
/^[-+]?(\d*\.?\d+|\d+\.)$/

Trimming a string

if (!String.prototype.trim) {
    String.prototype.trim = function(){
        return this.replace(/^\s+|\s+$/g, '');
    }
}

What we learned

  • \b = word boundary = between \w and \W|^|$
  • \B = non-word boundary = between \w and \w or \W and \W
  • Assertions are zero width

What we learned

  • (?=a) = followed by a, which can be any regex
  • (?!a) = NOT followed by a
  • No lookbehind in ECMAScript :(

Lookahead hacks

Intersection

// A 6+ letter password with at least:
// one number, one letter and one symbol
/^(?=.*\d)(?=.*[a-z])(?=.*[\W_]).{6,}$/i

Subtraction

// Any number that’s NOT divisible by 50
/\b(?!\d+[50]0)\d+\b/

Negation

// Anything that doesn’t contain "foo"
/^(?!.*foo).+$/

  • \N = Nth backreference

Precision vs practicality

False positives vs false negatives

Keep it simple

Even if it means not using regexes at all

For performance

  • Avoid greedy quantifiers
  • Don’t forget anchors (^ and $)
  • Be as specific as possible
  • Prefer non-capturing groups (?:)
  • Minimize backtracking

RegExp#exec

/a(b)/.exec('abba'); // ["ab", "b"]
var reg = /a(b)/g, bar = 'abba', matches = [], m;
while (m = reg.exec(bar))
  matches.push({
    index: m.index, // reg.lastIndex-m[0].length+1
    length: m[0].length,
    groups: m
  });

String#match

>>> 'abcdef'.match(/(\w){3}/);
["abc", "c"] // Same as /(\w){3}/.exec('abcdef')
>>> 'abcdef'.match(/(\w){3}/g);
["abc", "def"]
>>> 'ab'.match(/(\w){3}/g);
null

String#replace

>>> 'blabla'.replace('b', ''); // "labla"
>>> 'blabla'.replace(/b/g, ''); // "lala"
>>> 'blabla'.replace(/(\w)/g, '$1$1'); // "bbllaabbllaa"
>>> 'border-radius'.replace(/-(\w)/g, function($0,$1){
	return $1.toUpperCase();
}); // "borderRadius"

Tools

XRegExp

By Steven Levithan

RegExp wrapper that adds awesomeness like:

  • Comments in regexes
  • Flag to turn off automatic capturing
  • Flag to ignore whitespace
  • Named capturing groups
  • Unicode-aware predefined classes
  • Smooths out browser bugs
  • Plugins for even more awesomeness

Regexpal

By Steven Levithan

Simple but powerful regexp tester

  • Highlights regex syntax and matches as you type
  • Includes a quick reference for regex syntax
  • Knows about syntax that doesn’t work correctly in some browsers, and marks it as an error
  • Pure JavaScript, so you can save it and use offline
